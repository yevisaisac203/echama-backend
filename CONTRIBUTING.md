# Contributing to echama-backend

First off, thank you for considering contributing! It’s people like you who make this tool better for everyone.

##  Getting Started
1. **Fork** the repository.
2. **Clone** your fork: `git clone https://github.com/yevisaisac203/[repo-name].git`
3. **Create a branch** for your feature: `git checkout -b feature/amazing-feature`

##  Development Standards
To maintain high-quality code, please follow these guidelines:
- **Backend:** Follow PEP 8 styles for Python/Django.
- **Documentation:** Ensure all new functions have clear docstrings.
- **Commits:** Use descriptive commit messages (e.g., `feat: add Daraja API integration`).

##  Pull Request Process
1. Ensure your code passes all tests.
2. Update the README if you changed any functionality.
3. Your PR should be linked to an existing Issue.
4. Once submitted, a maintainer will review your code within 48 hours.

##  Reporting Issues
Use the GitHub "Issues" tab to report bugs or suggest features. Please include:
- A clear description of the bug.
- Steps to reproduce the error.
- Expected vs. Actual results.
